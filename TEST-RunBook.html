üöÄ1. Project Overview

NotifyBot is an advanced automated email batch sender designed for enterprise-scale email campaigns. It provides sophisticated filtering, template substitution, and batch processing capabilities with comprehensive logging and validation.

NotifyBot, our smart, scalable solution for email automation. Whether you're sending a handful of emails or thousands, NotifyBot makes it effortless, safe, and efficient. It supports both single and multi modes, with dry-run and signature capabilities.

  • üìß Batch Email Sending: Deliver emails in customizable batches with controlled delays

‚öôÔ∏è3. Operation Modes

üéØ Single Mode

Purpose: Send ONE email to MULTIPLE recipients

  • All recipients receive the same email content
  • Supports TO, CC, and BCC recipients
  • Optional filtering from inventory.csv
  • Perfect for announcements, newsletters, broadcasts

üéØ Multi Mode

Purpose: Send MULTIPLE personalized emails

  • Each filter condition generates a separate email
  • Template substitution with field values
  • Personalized content per recipient group
  • Perfect for targeted campaigns, departmental updates
Mode Priority: CLI argument ‚Üí mode.txt file ‚Üí default (single)

üéØ Single Mode Requirements

  • subject.txt Required
  • body.html Required
  • from.txt Required
  • approver.txt Required
  • to.txt Optional
  • cc.txt Optional
  • bcc.txt Optional
  • filter.txt Optional
  • additional_to.txt Optional
Note: At least ONE recipient source required (to.txt, filter.txt, additional_to.txt, cc.txt, or bcc.txt)

üîÑ Multi Mode Requirements

  • subject.txt Required
  • body.html Required
  • from.txt Required
  • approver.txt Required
  • filter.txt Required
  • inventory.csv Required
  • field.txt Optional
  • cc.txt Optional
  • bcc.txt Optional
  • additional_to.txt Optional
Note: Creates multiple personalized emails based on filter conditions

üß™4. Dry-Run Option

Safe Testing: The dry-run mode allows you to test your email campaigns safely before sending to actual recipients.

How Dry-Run Works:

  • Recipients: Emails sent only to approvers (from approver.txt)
  • Subject: Automatically prefixed with "DRAFT -"
  • Content: Includes draft banner with recipient statistics
  • Validation: Full validation runs (files, filters, field names)
  • Statistics: Shows what would happen in live mode
python notifybot.py --base-folder emails --dry-run
‚ö†Ô∏è Important: In dry-run mode, if to.txt already exists, it won't be overwritten. Delete to.txt to see fresh filter results.

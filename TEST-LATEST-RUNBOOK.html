Flexible Notification Types

Choose the optimal approach for your email notifications with our dual-mode architecture designed for different business scenarios.

🗂️ Directory Structure Guide

Professional file organization and campaign management reference

🗂️ Complete Directory Structure

/notifybot/ # Root directory ├── basefolder/ # Campaign base directory │ ├── campaign1/ # Example campaign folder │ │ ├── subject.txt # Email subject line │ │ ├── body.html # Email body content (HTML) │ │ ├── from.txt # Sender email address │ │ ├── approver.txt # Approver emails for dry-run │ │ ├── mode.txt # Mode: "single" or "multi" (optional) │ │ ├── to.txt # Direct recipient list (semicolon-separated) │ │ ├── cc.txt # CC recipients (semicolon-separated) │ │ ├── bcc.txt # BCC recipients (semicolon-separated) │ │ ├── filter.txt # Filter conditions for inventory │ │ ├── field.txt # Field names for substitution (multi mode) │ │ ├── additional_to.txt # Additional recipients to merge │ │ ├── attachment/ # Email attachments folder (optional) │ │ │ ├── document1.pdf │ │ │ ├── report.xlsx │ │ │ └── ... │ │ ├── images/ # Email embedded images (optional) │ │ │ ├── logo.png │ │ │ ├── banner.jpg │ │ │ └── ... │ │ └── recipients/ # Generated recipient files (auto-created) │ │ ├── filter_001_department_sales.txt │ │ ├── cc_recipients.txt │ │ ├── multi_mode_summary.txt │ │ └── all_unique_recipients.txt │ ├── campaign2/ # Another campaign folder │ └── campaign3/ # More campaigns... ├── inventory/ # Global inventory directory │ └── inventory.csv # Master contact database ├── logs/ # Log files directory │ └── notifybot.log # Application logs (auto-created) ├── signature.html # Global email signature (optional) └── notifybot.py # Main application script

Note: Creates multiple personalized emails based on filter conditions

📋 File Descriptions

📄 Core Configuration Files

  • subject.txt: Email subject line (supports template placeholders in multi mode)
  • body.html: Email body content in HTML format (supports template placeholders)
  • from.txt: Sender email address
  • approver.txt: Email addresses to receive draft emails during dry-run mode
  • mode.txt: Contains "single" or "multi" to set operation mode

👥 Recipient Management Files

  • to.txt: Direct recipient email addresses (semicolon-separated)
  • cc.txt: CC recipient email addresses (semicolon-separated)
  • bcc.txt: BCC recipient email addresses (semicolon-separated)
  • additional_to.txt: Additional recipients to merge with main list
  • filter.txt: Filter conditions to select recipients from inventory.csv
  • field.txt: Field names for template substitution (multi mode only)

🗃️ Data & Asset Files

  • inventory.csv: Master contact database with headers for filtering
  • signature.html: Global email signature (automatically appended to all emails)
  • attachment/ folder: Contains files to attach to emails
  • images/ folder: Contains images to embed in email HTML
  • recipients/ folder: Auto-generated files with recipient details

⚠️ Configuration Guidelines

🔒 Directory Path Requirements

  • All campaign folders must be inside /notifybot/basefolder/
  • Inventory file must be at /notifybot/inventory/inventory.csv
  • Global signature file at /notifybot/signature.html
  • Logs are automatically created at /notifybot/logs/notifybot.log

✅ Field Validation Rules

  • All field names in filter.txt must exist in inventory.csv headers
  • All field names in field.txt must exist in inventory.csv headers (multi mode)
  • Field validation occurs before email sending to prevent errors
  • Case-sensitive field name matching is enforced

🎯 Mode Selection Priority

  1. CLI argument: --mode single or --mode multi (highest priority)
  2. mode.txt file: Contains "single" or "multi"
  3. Default: "single" mode if not specified anywhere

🚀 Quick Start Examples

📧 Basic Single Mode Campaign

# Create basic single mode campaign
mkdir -p /notifybot/basefolder/my-campaign
cd /notifybot/basefolder/my-campaign

# Create required files
echo "Monthly Newsletter" > subject.txt
echo "admin@company.com" > from.txt
echo "approver@company.com" > approver.txt
echo "<h1>Hello!</h1><p>This is our monthly newsletter.</p>" > body.html
echo "user1@example.com;user2@example.com" > to.txt

# Run dry-run to test
python /notifybot/notifybot.py --base-folder my-campaign --dry-run

# Run live campaign
python /notifybot/notifybot.py --base-folder my-campaign --force

🔄 Advanced Multi Mode Campaign

# Create multi mode campaign with personalization
mkdir -p /notifybot/basefolder/personalized-campaign
cd /notifybot/basefolder/personalized-campaign

# Create required files
echo "Quarterly Report for {department}" > subject.txt
echo "admin@company.com" > from.txt
echo "approver@company.com" > approver.txt
echo "<h1>Report for {department}</h1><p>Region: {region}</p>" > body.html
echo 'department="sales"' > filter.txt
echo -e "department\nregion" > field.txt

# Ensure inventory exists with required columns
# /notifybot/inventory/inventory.csv should have: department,region,email

# Test with dry-run first
python /notifybot/notifybot.py --base-folder personalized-campaign --mode multi --dry-run

# Execute live campaign
python /notifybot/notifybot.py --base-folder personalized-campaign --mode multi --force
